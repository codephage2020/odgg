## Design Context

### Users
- **Primary**: Mixed team — data engineers lead modeling, analysts participate in review
- **Context**: Daily dimensional modeling workflow, collaborative model review sessions
- **Job to be done**: Build Kimball-style star schemas from source databases with AI assistance, generate production-ready DDL/ETL/dbt code

### Brand Personality
**Professional + Precise + Quiet**
- Tool-grade interface that stays out of the way
- Precision over decoration — every element earns its place
- Calm confidence, not flashy or attention-seeking

### Aesthetic Direction
- **Tone**: Refined, utilitarian workbench — like Linear meets Figma's canvas
- **Reference**: [Cube.dev](https://cube.dev) — enterprise data tooling with clean, accessible UI; BI-inspired layouts with data type iconography; semantic color tokens
- **Anti-reference**: Retool / low-code drag-and-drop cheapness; generic AI dashboard slop (cyan gradients, glowing cards)
- **Theme**: Adaptive dual mode (light + dark), system-preference aware
- **Typography**: Professional sans-serif display + clean body font, monospace for code/data
- **Color**: Muted, purposeful palette — semantic colors for data types (dimensions, measures, facts), restrained accent usage

### Design Principles
1. **Canvas is king** — the data model diagram is the hero; everything else supports it
2. **Quiet confidence** — professional tools don't shout; subtle transitions, restrained color, precise spacing
3. **Data-type semantics** — consistent visual language for dimensions, measures, facts, grain across all surfaces
4. **Progressive density** — start simple, reveal complexity through interaction; analysts see clarity, engineers see depth
5. **Dual-persona readability** — works for engineers writing SQL and analysts reviewing business logic
